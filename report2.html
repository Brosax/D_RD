SESIP-001
Buffer Overflow in Nonce Storage
CRITICAL | Confidence: HIGH | File: src/storage.c:72
Description

The function storage_store_last_nonce() copies sizeof(g_last_nonce) + 1 bytes from an untrusted source into a fixed-size buffer, causing a one-byte buffer overflow.

7071#if INJECT_VULNS 72 memcpy(g_last_nonce, nonce, sizeof(g_last_nonce) + 1u); 73#else
Attack Scenario

An attacker who can control the nonce input can overflow g_last_nonce by one byte, potentially corrupting adjacent memory structures and bypassing replay protection.

Impact

Memory corruption, potential bypass of attestation replay protection, denial of service.

Remediation

Use nonce_len instead of sizeof(g_last_nonce) + 1u, and validate nonce_len equals the expected nonce size before copying.

SESIP-002
Buffer Overflow in Communication Packet Processing
CRITICAL | Confidence: HIGH | File: src/comm.c:62
Description

The comm_process_packet() function copies payload data into a fixed-size local buffer without validating that the payload length does not exceed the buffer size.

6061#if INJECT_VULNS 62 memcpy(local, payload, payload_len); 63#else
Attack Scenario

A remote attacker can send a packet with payload_len > LOCAL_BUF_SZ (64 bytes), causing a stack buffer overflow. This can lead to stack corruption, code execution, or crashes.

Impact

Remote code execution via stack buffer overflow, denial of service, potential bypass of security controls.

Remediation

Add bounds checking before the memcpy: if (payload_len > sizeof(local)) return FW_ERR_INVALID_ARG;

SESIP-006
Weak Replay Protection in Attestation
MEDIUM | Confidence: HIGH | File: src/attestation.c:35
Description

The replay protection in attestation_generate_report() only compares the first 4 bytes of the challenge nonce instead of the full 16 bytes.

3334 if (memcmp(challenge, g_last_challenge, 4) == 0) {
Attack Scenario

An attacker can replay a previous attestation challenge if it shares the same first 4 bytes as a new challenge. This reduces the effective replay protection from 2^128 to 2^32 possibilities.

Impact

Reduced replay protection strength from 128 bits to 32 bits, making replay attacks feasible.

Remediation

Compare the full ATTEST_NONCE_LEN (16 bytes) of the challenge: memcmp(challenge, g_last_challenge, ATTEST_NONCE_LEN)
